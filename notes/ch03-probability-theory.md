## 一、正态分布 vs 对数正态 — 金融的核心假设

### 1.1 为什么对数正态是金融的标准模型

```
MIT讲义原文:
"To model the stock market, it is more reasonable to assert that the rate
of change of the stock price has normal distribution (compared to the
stock price itself having normal distribution)."
```

翻译：**价格的变化率**（对数收益率）服从正态 → 价格本身服从对数正态。

数学上：
```
X ~ N(μ, σ²)  →  Y = e^X ~ LogNormal(μ, σ²)

f_Y(y) = (1/(y·σ·√(2π))) · exp(-(ln y - μ)²/(2σ²)),  y > 0
```

### 1.2 对数正态的关键性质

```
E[Y] = exp(μ + σ²/2)
Var[Y] = (exp(σ²) - 1) · exp(2μ + σ²)
```

**关键洞见**：对数正态的均值不是 e^μ，而是 e^(μ+σ²/2)。波动率越高，均值越高——这在直觉上不成立（高波动应该高风险），但数学上确实如此。这解释了为什么高 σ 的资产长期平均收益可能更高——不是"风险补偿"，而是**对数正态的数学特性**。

### 1.3 你的系统应用：BTC 收益率是否真的服从对数正态？

```
你的BTC/ETH/SOL日收益率数据 (179个样本):
- BTC: 年化波动率 47.9%
- ETH: 年化波动率 64.5%
- SOL: 年化波动率 65.1%

如果真服从对数正态：
- BTC日收益的峰度应 ≈ 3 (正态峰度)
- 偏度应 ≈ 0 (对称)

但加密货币真实数据的峰度通常 >> 3 (肥尾)
这意味着对数正态假设严重低估了极端事件概率
→ 你的止损线和熔断线如果用正态假设，会低估风险
```

### 1.4 指数族分布 — 为什么MLE在这里很好用

MIT 讲义提到指数族：
```
f^(θ)(x) = h(x)·c(θ)·exp(Σ w_i(θ)·t_i(x))
```

对数正态是指数族的一员。指数族有"充分统计量"——t_i(x) 包含了关于参数 θ 的全部信息。

**对你的系统**：如果收益率真的服从对数正态，那么用 ln(收盘价) 的均值和方差就足够了。但如果不是（你的数据多半不是），需要更复杂的分布。

## 二、矩母函数 (MGF) — 比你想象的更重要

### 2.1 定义

```
M_X(t) = E[e^(tX)]
```

第 k 阶矩 = M_X^(k)(0)（对 t 求 k 阶导数后取 t=0）

### 2.2 为什么对数正态没有MGF？

MIT 讲义原文：
> "the log-normal distribution does not have a moment generating function.
> In fact, E[e^(tX)] does not exist for all t ≠ 0."

**这是极其重要的一点**：对数正态分布没有矩母函数！因为 e^(tX) 的尾部太重，积分发散。

**对你的系统的直接影响**：
1. 中心极限定理要求 MGF 存在 → 对数正态不满足
2. 如果你的收益率服从对数正态，CLT 的结论不严格适用
3. T 检验在小样本下可能严重不靠谱
4. **这就是为什么你的 bet_4 和 bet_5 的 t 统计量都不显著**

### 2.3 MGF 唯一性定理的陷阱

```
定理2.2(i): 如果 M_X(t) = M_Y(t) 对所有 t → X 和 Y 同分布
```

但讲义警告 (page 4):
> "It does not imply that all two random variables with same moments
> have the same distribution"

两个分布可以有相同的所有矩但不相同——这叫"矩不确定问题"(moment indeterminacy)。对数正态就是例子。

## 三、大数定律 — 你的回测为什么可能运气好

### 3.1 弱大数定律

```
X̄_n = (X₁ + ... + Xₙ)/n,  E[X̄_n] = μ,  Var[X̄_n] = σ²/n

→ P(|X̄_n - μ| ≥ ε) ≤ σ²/(nε²) → 0 as n → ∞
```

### 3.2 赌场类比 (MIT讲义 Example 3.2)

MIT用赌场来说明大数定律：
- 轮盘赌：赌场优势 ~1-5% per game
- 方差 σ 远大于均值
- 短期看起来"任何事都可能发生"（玩家能赢）
- 长期大数定律保证赌场赚钱

**这完美描述了你网格策略的问题**：
- 网格的"优势"（bid-ask spread）≈ 0.1-0.5% per trade
- 方差（单笔盈亏波动）远大于均值
- 短期：价格突破区间 → 你被止损
- 长期：只要网格捕获均值回复 → 应该能赚钱

**但关键是：你需要多少笔交易才能让 LLN 生效？**

```
Var[X̄] = σ²/n

你的网格 σ_per_trade ≈ 2-5%（单笔波动）
n = 100笔 → SE ≈ 0.2-0.5%
n = 1000笔 → SE ≈ 0.06-0.16%
n = 10000笔 → SE ≈ 0.02-0.05%

如果你的期望收益 per trade = 0.1%，你需要 n >> (σ/0.001)² 笔才能有统计显著性
例如 σ=3% → n >> 900 笔
```

**你这179天里网格成交了多少笔？如果不够900笔，LLN还没生效。**

## 四、中心极限定理 — T检验的基础和陷阱

### 4.1 CLT

```
Z_n = √n · (X̄_n - μ) → N(0, σ²)  as n → ∞
```

CLT告诉你：不管原始分布是什么形状（只要方差有限，MGF存在），样本均值的分布在 n→∞ 时趋近正态。

### 4.2 CLT的三大前提 — 你的数据满足几个？

```
1. i.i.d. (独立同分布)     ❌ 收益率有自相关 (第一章诊断已验证)
2. 有限的方差             ⚠️ 加密货币有肥尾，方差可能无限
3. MGF存在               ❌ 对数正态没有MGF
```

**你的收益率几乎不满足CLT的任何前提**。

### 4.3 对第一章 Beta/Alpha 分析的影响

第一章里用的 t-test 假设 CLT 成立。但你的 BTC/ETH/SOL 数据有：
- 自相关 (残差自相关=0.6 for BTC 20d)
- 潜在的非正态肥尾
- 179个样本可能不够大

**这意味着你的 t-stat 和 p-value 可能不可靠**。这不是代码写错了，而是统计模型的假设不成立。

## 🔥 实践对照：币种量化系统 vs 概率论

### 对照1：LLN → 你的样本量够不够？

```
bet_8: BTC/ETH/SOL 网格，179天数据
假设成交率 2笔/天 → 约 358 笔交易

σ_per_trade ≈ 3%（网格间距内的波动）
期望收益 per_trade ≈ 0.3%（价差的 ~一半）

需要的n: n > (0.03/0.003)² = 100 → 理论够了
但实际: 358笔 → SE ≈ 0.16%
t_stat ≈ 0.3%/0.16% ≈ 1.88 → 边缘显著

结论: 179天的样本量勉强够，但远不充裕
```

### 对照2：对数正态 → 你的止损失效分析

```
对数正态的真期望变CVaR:
95%置信 CVaR = E[X|X ≤ VaR_0.05] 

如果你的止损线基于正态假设的 ±2σ：
止损触发概率(正态) = 2.3% (95%误差)
止损触发概率(真实,肥尾) = 可能5-8% (如果峰度=6)

这意味着你的止损被触发的频率是你预期的2-3倍！
```

### 对照3：MGF不存在 → 蒙特卡洛的隐患

```
你的回测/模拟隐含假设: 收益率分布可由前两阶矩(均值+方差)充分描述
但 MGF不存在 → 高阶矩(偏度+峰度)独立携带重要信息
→ 你需要更多数据才能准确估计极端事件的概率
→ 简单的历史模拟(用179天直接重采样)比参数化MC更可靠
```
